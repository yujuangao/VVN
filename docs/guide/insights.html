VVN Insights

Design standards for VVN Insight posts — narrative data stories with charts, maps, and tables.

ImportantKey Rules at a Glance
  • Lead with the finding — state the most important result in the first paragraph
  • Reference every figure in the text before it appears: “Figure 1 shows…”
  • Search for [ before publishing — no placeholder text in the final output
  • Insights are for reading, not exploration — use the Dashboard for interactive filtering

Overview

A VVN Insight is a Quarto HTML document that tells a data story. It combines a narrative with charts, an interactive map, and a summary table — all styled to the VVN design standard. Insights are designed for reading, not exploration.

Use create_vvn_story() from the vvnthemes library to scaffold this structure with one command.

To scaffold and build a VVN Insight, see the VVN Insight Library →

Example

VVN Insights Example — Broadband Access Across Virginia’s Counties

1
Hero banner — Full-width maroon Background: VT Maroon #861F41. Contains “VVN INSIGHTS” label in small caps, the full insight title in white bold (headline case), and a one-sentence description of the topic and geography. Subtitle reads “Visualizing Virginia Numbers · Virginia Tech” at 70% white opacity.
2
Headline — White, headline case, bold States the topic and scope. Example: “Broadband Access Across Virginia’s Counties.” Should communicate the subject immediately.
3
Publication date and source tag Publication date and primary data source appear below the headline. Include the dataset name and vintage year.
4
Choropleth map — County level Primary geographic visualization. Uses maroon_seq palette. Interactive with tooltip showing county name and value. Referenced in the surrounding text.
5
KPI callout — Orange accent Key statistics highlighted in orange callout boxes. Use for the most important numbers only. Example: “76% of Virginia households have broadband.”
6
Narrative — Left-aligned, structured sections Body text in Charcoal #3D3D3D. Short paragraphs, 3–4 sentences. Clear section headings in sentence case. Every figure referenced in the text before it appears.
7
Source line — Required on every figure Every chart, map, and table must have a source line below it. Format: “Source: [Agency, Dataset, Year]. [Percentage note or additional detail].”

Document Structure

Every VVN Insight must follow this section order:

# Section Required Content Requirements
1 Hero Banner Yes “VVN INSIGHTS” label · headline (headline case) · one-sentence description of topic and geography · “Visualizing Virginia Numbers · Virginia Tech” in subtitle
2 Key Finding Yes 1–2 sentences · state the metric, geography, time period, and direction of change · use blockquote (>) or :::{.callout-important} · bold the key number
3 Background Yes Why this topic matters for Virginia · policy relevance · the specific question this Insight addresses · intended audience (policymakers, advocates, community organizations)
4 Data & Methods Yes Data source table (Source, Geography, Period, Variable) · callout noting caveats or comparability issues
5 Findings Yes Subsections by theme · each subsection: heading (sentence case) + 1–3 paragraphs + at least one figure · minimum: 2 charts, 1 map, 1 summary table · every figure referenced in text before it appears
6 Discussion Yes Interpretation of findings · connection to prior research or policy context · equity implications · limitations (data gaps, geography, causal claims)
7 Conclusion Yes 2–3 sentence takeaway · policy implication or future work direction · VVN contact info and Virginia Tech attribution
