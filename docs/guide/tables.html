Tables

Style rules for VVN summary tables, data tables, and regression tables.

Overview

VVN tables follow strict formatting rules for font styles, number alignment, and cell shading. These conventions make VVN tables look specific and consistent across all products. Every table must have a source line.

For R functions, gt styling, and implementation, see the Table Library →

Required elements

# Element Standard Library function
1 Title Headline case, bold, one line. Short and active. Example: “Virginia Counties at a Glance.” vvn_table()
2 Subtitle Sentence case (optional). States geography and year. Example: “Virginia, 2022 estimates.” vvn_table()
3 Column headers VT Maroon #861F41 fill, white bold text. Sentence case for standard columns; headline case for spanner headings. vvn_table()
4 Alternating rows Even rows use light gray #F7F7F7. No internal borders by default. No empty rows or columns. vvn_table()
5 Cell alignment Numbers: decimal-aligned, right. Text: left. Use consistent decimal places within each column. vvn_table()
6 Source & notes Source required below every table. Notes define abbreviations, methodology, and significance. Missing values: em dash (—), never blank. vvn_table()

Design standards

Number formatting

Value type Format
Percentages 72.4% — one decimal place, percent sign
Dollar amounts $48,200 — no decimals for whole dollars
Counts 1,234 — comma separator, no decimals
Index values 0.743 — three decimal places
P-values < 0.001 or 0.023 — never 0.000

Column alignment

  • Numbers: right-aligned, decimal-point aligned
  • Text categories: left-aligned
  • Header text: centered over numeric columns, left-aligned over text columns

Missing data

  • Always use an em dash (—) for missing or suppressed values
  • Define suppression rules in the notes below the table
  • Never leave a cell blank

Spanner headers

Use spanner headings (headline case) to group related columns. Keep the spanner label short — 1–3 words.
