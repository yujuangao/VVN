Dashboard

Layout and component standards for VVN Interactive Data Visualizations (R Shiny).

ImportantKey Rules at a Glance
  • Maroon navbar full width across the top; sidebar is 260 px wide and light gray
  • Three KPI cards per row — one card per key metric, no more
  • Every filter set must have a corresponding Apply Filters button
  • All charts and tables inside the dashboard follow the same title/source/capitalization rules as standalone figures

Overview

The Interactive Data Visualizations dashboard is an R Shiny application that provides an exploratory interface for VVN data. It uses a maroon navigation bar, sidebar filters, KPI summary cards, charts, an interactive county map, and a downloadable data table.

Use create_vvn_dashboard() from the vvnthemes library to scaffold this structure with one command.

To scaffold and build a dashboard, see the Dashboard Library →

Example

Dashboard Example — Broadband Access in Virginia

1
Navbar — VT Maroon, full width Background: maroon #861F41. Project name in white bold text. Tab links in white at 90% opacity. Orange bottom border (3px). Navigation tabs clearly labeled.
2
KPI Cards — Three per row Large metric value, short label, and optional trend indicator (up/down/flat). Use maroon, orange, or navy — one color per card, consistently applied across the dashboard.
3
Sidebar Filters — 260px, light gray Background: #F7F7F7. Contains dropdowns, range sliders, and Apply / Reset buttons. Always label each filter clearly and include a tooltip explaining what it controls.
4
Choropleth Map — maroon_seq palette Full-panel interactive Leaflet map on the Map tab. County-level fill. Tooltip shows county name and value. Legend inside panel.
5
Charts — VVN themes throughout All charts follow VVN standards: white background, maroon titles, VVN color palettes, source line in caption. Apply vvnthemes → to all.
6
Footer — Visualizing Virginia Numbers · Virginia Tech Thin footer bar: white text on maroon background. Include VVN name and Virginia Tech affiliation.

Tab Structure

The dashboard template has four tabs. Each tab has a defined purpose and required elements.

Tab 1 — Overview

The landing page. Combines sidebar filters, KPI cards, and charts into a single exploratory view.

Element Details
Sidebar filters At least one dropdown + one range slider. Apply button (maroon, full width) and Reset button (outline) required.
KPI cards 3 cards (maroon, orange, navy). Each card needs a value, a short label, and an optional directional trend indicator (up / down / flat) with a comparison period.
Trend chart Line chart of the primary metric over time. Title (headline case), subtitle (sentence case), and source line required.
Comparison chart Bar or column chart of the latest-year comparison across categories. Title, subtitle, and source line required.
Distribution chart Histogram or boxplot of the primary metric distribution. Title, subtitle, and source line required.

Tab 2 — Map

Full-panel county-level interactive choropleth map of Virginia.

Element Details
Choropleth map Virginia county-level Leaflet map. Projection: WGS84 (CRS 4326). Use maroon_seq for sequential data; diverging for above/below-average data. See Map Library →
Legend Value breaks clearly labeled; placed inside the map panel.
Tooltip County name, metric value, and data range on hover. Format values consistently (e.g., “45.2%”, “$129,456”).
Variable selector Allow users to switch the mapped variable. The map panel header must name the variable displayed.

Tab 3 — Data

Downloadable, searchable table of the underlying data.

Element Details
Interactive data table Paginated, searchable, sortable DT table. Responds to sidebar filters. Include only relevant columns — no raw IDs or processing artifacts. See Table Library →
CSV download Download button for the filtered dataset.
Excel download Download button for the filtered dataset.
Column formatting Numbers right-aligned; text left-aligned. Column headings use sentence capitalization. Apply color-bar to the primary numeric column for visual scanning.

Tab 4 — About

Project documentation for transparency and reproducibility.

Element Details
Project description 1–2 paragraphs explaining the dashboard’s purpose and audience.
Data sources Bulleted list with full citations for all data used.
Methods Brief description of any data processing or analytical decisions.
Contact VVN contact email or link.

Filter Panel Requirements

Filter Panel Example — Sidebar Filter Controls

1
Filter header — Maroon #861F41 “Filters” label in white bold text. Include a Clear All reset link and funnel icon. Full-width maroon header bar.
2
Dropdown — Clear label + info tooltip Each filter has a bold label above the input and an ⓘ icon that opens a tooltip explaining what the filter does and how data updates annually.
3
Info tooltip — Explain each filter Always include a tooltip on each filter explaining what it controls, what the choices mean, and when the data was last updated.
4
Apply Filters button — Maroon, full width Primary action button. Always pair with a Reset button directly below in outline style. Use Interactive Component Library →
