Interactive Data Visualizations

Layout and component standards for VVN Interactive Data Visualizations (R Shiny).

Interactive Data Visualizations is a VVN dashboard — an interactive R Shiny web application. It lets users filter and explore VVN data on their own — choosing regions, years, and topics while charts, KPI cards, a county map, and a data table all update in real time.

Use it when users need to explore data. For a fixed narrative with a single finding, use a VVN Insights report instead.

Project structure

create_vvn_dashboard() scaffolds this folder structure automatically:

my-dashboard/
├── app.R                    ← Main Shiny app; load data, display charts, reactive UI
├── scripts/
│   └── analysis.R           ← Build all ggplot2 charts here; source this first
├── figures/                 ← Auto-created when analysis.R runs; holds numbered PNGs
│   ├── 01_trend_statewide.png
│   ├── 02_trend_by_region.png
│   └── ...
├── www/
│   └── vvn.css              ← VVN brand styles (do not edit)
└── data/
    ├── raw/                 ← Original source files (do not edit)
    │   ├── county_data.csv
    │   └── demographics.csv
    └── processed/           ← Cleaned, analysis-ready files (load in app.R)
        └── my_data.csv
  • app.R — displays static chart PNGs from figures/ alongside reactive KPI cards, sidebar filters, data table, and optional map.
  • scripts/analysis.R — all chart code lives here. Source the script to auto-number and save PNGs to figures/.
  • data/raw/ — place original downloaded files here and never modify them.
  • data/processed/ — save cleaned, analysis-ready files here; load these in app.R.
  • www/vvn.css — compiled VVN brand stylesheet; do not edit.

Required libraries

Install all packages before running create_vvn_dashboard().

app.R (dashboard runtime)

install.packages(c("shiny", "bslib", "bsicons", "dplyr", "readr"))
# Optional — uncomment in app.R when needed:
# install.packages(c("leaflet", "sf", "DT", "gt"))

scripts/analysis.R (chart building)

install.packages(c("ggplot2", "dplyr", "tidyr"))
# Optional — for saving Leaflet maps as PNG:
# install.packages("mapshot2")

vvnthemes is required in both files and is loaded automatically by the scaffold.

How to create a dashboard

1
Scaffold the projectRun one command in R. The library creates the full folder structure, all required files, VVN styles, and placeholder content — the dashboard runs immediately with no setup.
Dashboard Library →
2
Load your data in app.RUncomment and fill in the data loading lines near the top of app.R. Load your CSV directly from data/processed/ or data/raw/ using readr::read_csv(). No separate data file is needed.
3
Build figures in scripts/analysis.ROpen scripts/analysis.R. Uncomment the chart types you need from the gallery and fill in your column names and labels. Source the script — figures are auto-numbered 01_, 02_, … and saved to figures/. Every chart must have a title, subtitle, and vvn_source() line. See Charts and Maps for full rules.
4
Wire charts and components in app.RIn each renderUI block, update the filename to match the PNG saved in figures/. Uncomment and fill in KPI cards, sidebar filters, and the data table. Every filter needs a bold label, a tooltip, and a default value. Always pair Apply Filters with a Reset button. See Filter Controls.
5
DeployRun rsconnect::deployApp() from the project root. Check the Publishing Checklist before deploying.

Required elements

Every VVN Dashboard must include all six elements. Use the vvnthemes library functions to build each one.

# Element Standard Library function
1 Navbar VT Maroon #861F41 background, white text. Active tab: VT Orange bottom border, 2 px. Bootstrap 5 via bslib. vvn_bs_theme()
2 KPI Cards Max 3 per row. Large value, short label, optional trend. One accent color per card — maroon, orange, or navy. vvn_kpi_card()
3 Sidebar Filters 260 px wide, #F7F7F7 background. Bold label on every filter. Apply + Reset buttons required. vvn_filter(), vvn_slider(), vvn_button()
4 Choropleth Map County-level Leaflet map. maroon_seq palette. Tooltip with county name and value. Legend inside panel. vvn_map_style()
5 Charts Built in scripts/analysis.R, saved as PNGs to figures/. VT Maroon titles, VVN palettes, vvn_source() required. theme_vvn(), vvn_source(), vvn_save()
6 Footer White text on dark navy #222C3D background. Shows “Visualizing Virginia’s Numbers · Virginia Tech”. create_vvn_dashboard()

Design standards

Navbar

  • Background: VT Maroon #861F41, foreground white — set via navbar_options = bslib::navbar_options(bg = "#861F41", fg = "#FFFFFF")
  • Theme: Bootstrap 5 via vvn_bs_theme() — no manual CSS needed
  • Project name: white, bold, sentence case
  • Active tab: VT Orange #E87722 bottom border, 2 px (from vvn.css)
  • Tab links: white — never colored or underlined unless active

KPI Cards

  • Left border accent: maroon, orange, or navy — one color per card, never mixed in the same row
  • Value: 2 rem, bold · Label: 0.82 rem, charcoal, sentence case
  • Maximum three cards per row

Sidebar Filters

  • Width: 260 px fixed · Background: #F7F7F7
  • Every filter must have: a bold label above, a tooltip (ⓘ) explaining what it controls, and a clear default
  • Apply Filters button: VT Maroon, full width · Reset button: outline style, directly below Apply

Map

  • Palette: maroon_seq for sequential data, diverging for change or comparison
  • County borders: thin white (0.5 px) · Tooltip: county name + formatted value + unit
  • Never use a non-VVN palette

Charts

Follow all Chart Standards. Source lines are required on every chart, even inside a dashboard.

Footer

  • Background: Dark navy #222C3D (from --ui-footer in vvn.css) · Text: white, 0.94 rem
  • Content: “Visualizing Virginia’s Numbers · Virginia Tech · vvn.vt.edu”
  • Rendered by tags$footer() in app.R — styled automatically by vvn.css
