Tables

Create a styled publication table in five steps.

For design rules, cell formatting, and border conventions, see Table Standards →

Create your first table

For most VVN projects, use vvn_table(). It handles all styling — maroon header, alternating rows, source line — automatically.

Step 1 — Install gt (once)

install.packages("gt")

gt is the R package that builds the underlying table. vvn_table() wraps it with VVN styling.

Step 2 — Load packages

library(vvnthemes)
library(gt)

Step 3 — Prepare your data

df <- data.frame(
  Region       = c("Urban Core", "Urban Fringe", "Suburban", "Small City", "Rural"),
  `2023 Value` = c(65.3, 61.8, 57.4, 52.1, 46.2),
  Change       = c("+4.1", "+3.9", "+3.5", "+3.2", "+3.0"),
  check.names  = FALSE
)

Sort rows before creating the table — gt displays them in the order they appear in your data frame.

Step 4 — Create the table

df |>
  gt() |>
  vvn_table(
    title       = "Economic Index by Region",
    subtitle    = "Virginia, 2023 estimates",
    source_note = "Source: VVN Example Data"
  )

The pattern is always: data |> gt() |> vvn_table(...). You must call gt() before vvn_table().

  • title — headline case; displayed in maroon above the table
  • subtitle — sentence case; geography and year
  • source_note — required on every published table

Step 5 — Format numbers (optional)

Chain gt formatting functions after vvn_table() to control decimal places, alignment, and separators:

df |>
  gt() |>
  vvn_table(
    title       = "Summary Table",
    source_note = "Source: [Dataset] · VVN Analysis"
  ) |>
  gt::fmt_number(columns = `2023 Value`, decimals = 1) |>
  gt::cols_align(align = "right", columns = -Region)

Common formatting functions:

Function What it does
gt::fmt_number(columns, decimals = 1) Round to N decimal places
gt::fmt_percent(columns, decimals = 1) Format as percentage
gt::fmt_currency(columns) Format as dollar amount
gt::cols_align(align = "right", columns) Right-align numbers
gt::tab_spanner(label, columns) Group columns under a header

Need an interactive table?

If you are building a Shiny dashboard where users need to search, sort, filter, or download data, use DT::datatable() instead of vvn_table().

install.packages("DT")   # once
library(DT)

DT::datatable(
  your_data,
  rownames   = FALSE,
  class      = "stripe hover compact",
  extensions = "Buttons",
  options    = list(
    pageLength = 15,
    dom        = "Bfrtip",
    buttons    = c("csv", "excel")
  )
)

In a Shiny app:

# Server
output$dt_table <- DT::renderDataTable({
  filtered() |>
    DT::datatable(rownames = FALSE, class = "stripe hover compact")
})

# UI
DT::dataTableOutput("dt_table")

Presenting model results

Use vvn_table_pvalues() to add automatic significance stars to a regression table.

library(gt)
library(broom)

model <- lm(mpg ~ wt + hp + cyl, data = mtcars)

tidy(model) |>
  mutate(across(where(is.numeric), \(x) round(x, 3))) |>
  gt() |>
  vvn_table(
    title       = "OLS Regression: Fuel Economy",
    subtitle    = "Dependent variable: miles per gallon",
    source_note = "Source: Motor Trend (1974). N = 32."
  ) |>
  vvn_table_pvalues(col_name = "p.value")

Significance notation: * p < 0.05 · ** p < 0.01 · *** p < 0.001 p < 0.001 cells are highlighted in maroon; p < 0.05 in orange.

Design rules

Rule Requirement
Title Headline case, concise
Column headings Sentence case
Numbers Right-aligned; consistent decimal places
Missing values Em dash — — never leave cells blank
Source Required on every table

Common mistakes

“could not find function ‘gt’” — You forgot library(gt). Run install.packages("gt") first, then library(gt).

“could not find function ‘vvn_table’” — You forgot library(vvnthemes).

vvn_table(df) gives an error. — The correct pattern is df |> gt() |> vvn_table(...). You must call gt() first.

Numbers have too many decimal places. — Add gt::fmt_number(columns = my_col, decimals = 1) after vvn_table().

Table fails VVN review. — You forgot source_note. It is required on every published table.

See Reference Manual → for full parameter documentation on vvn_table() and vvn_table_pvalues().

## Common mistakes

**"could not find function 'gt'"** — You forgot `library(gt)`. Run `install.packages("gt")` first, then `library(gt)`.

**"could not find function 'vvn_table'"** — You forgot `library(vvnthemes)`.

**`vvn_table(df)` gives an error.** — The correct pattern is `df |> gt() |> vvn_table(...)`. You must call `gt()` first.

**Numbers have too many decimal places.** — Add `gt::fmt_number(columns = my_col, decimals = 1)` after `vvn_table()`.

**Table fails VVN review.** — You forgot `source_note`. It is required on every published table.
