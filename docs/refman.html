Setup

set_vvn_defaults

Set theme_vvn() as the global ggplot2 theme and update default geom aesthetics for the session. Bars, lines, and points become VT Maroon; smooth lines become VT Orange. Call once at the top of every script.

Usage

set_vvn_defaults(style = c("standard", "map"), base_size = 12)

Arguments

style
"standard" applies theme_vvn(); "map" applies theme_vvn_map(). Default: "standard".
base_size
Base font size passed to the theme. Default: 12.

Value

Invisibly NULL. Called for its side effects — sets the ggplot2 global theme and geom color defaults.

Examples

library(vvnthemes)
set_vvn_defaults()

ggplot2::ggplot(mpg, ggplot2::aes(displ, hwy)) +
  ggplot2::geom_point()   # automatically VT Maroon, theme_vvn()

undo_vvn_defaults()       # revert when done

undo_vvn_defaults

Revert to ggplot2 factory defaults, undoing set_vvn_defaults().

Usage

undo_vvn_defaults()

Value

Invisibly NULL.

Examples

set_vvn_defaults()
# ... make VVN charts ...
undo_vvn_defaults()

Themes

theme_vvn

Standard VVN ggplot2 theme: white background, VT Maroon titles, light gray horizontal grid lines, and clean typography. Use for bar charts, line charts, histograms, and most non-map figures.

Usage

theme_vvn(base_size = 12, base_family = "sans", grid = "y")

Arguments

base_size
Base font size in points. Default: 12.
base_family
Font family. Default: "sans".
grid
Grid lines to display: "y" (horizontal only, default), "x" (vertical only), "both", or "none".

Value

A ggplot2::theme object.

Examples

library(ggplot2)

# Bar chart — horizontal grid (default)
ggplot(mpg, aes(class)) +
  geom_bar() +
  theme_vvn()

# Scatter plot — both axes
ggplot(mpg, aes(displ, hwy)) +
  geom_point() +
  theme_vvn(grid = "both")

# Larger base size for presentations
ggplot(mpg, aes(class)) +
  geom_bar() +
  theme_vvn(base_size = 16)

theme_vvn_map

Map theme for ggplot2 + sf static choropleth maps. Removes axes and ticks; adds a muted panel background and places the legend at the bottom.

Usage

theme_vvn_map(base_size = 11, base_family = "sans")

Arguments

base_size
Base font size in points. Default: 11.
base_family
Font family. Default: "sans".

Value

A ggplot2::theme object.

Examples

library(ggplot2); library(sf)

ggplot(va_counties_sf, aes(fill = pct_broadband)) +
  geom_sf(color = "#FFFFFF", linewidth = 0.3) +
  scale_fill_vvn_c("maroon_seq") +
  theme_vvn_map() +
  labs(title = "Broadband Access by County", fill = "Households (%)")

theme_vvn_minimal

Minimal VVN variant: no grid lines, no axes, white background. Use for small multiples, annotation-heavy figures, and inset charts.

Usage

theme_vvn_minimal(base_size = 12, base_family = "sans")

Arguments

base_size
Base font size in points. Default: 12.
base_family
Font family. Default: "sans".

Value

A ggplot2::theme object.

Examples

ggplot(df, aes(x, y)) +
  geom_col() +
  theme_vvn_minimal()

Color System

vvn_colors

Return one or more named VVN brand colors as a named character vector of hex codes.

Usage

vvn_colors(...)

Arguments

...
Color name(s) to return. Valid names: "maroon", "orange", "navy", "charcoal", "gray", "light_gray", "white", "maroon_dk", "maroon_lt", "orange_lt". If none provided, returns all colors.

Value

A named character vector of hex codes.

Examples

vvn_colors()                             # all brand colors
vvn_colors("maroon")                     # "#861F41"
vvn_colors("maroon", "orange", "navy")   # three primary colors

vvn_palette

Generate a vector of hex colors from a named VVN palette. Sequential and diverging palettes are interpolated when n exceeds the base palette size.

Usage

vvn_palette(palette = "main", n = NULL, reverse = FALSE, alpha = 1)

Arguments

palette
Palette name. Categorical: "main" (10 colors), "brand" (maroon / orange / navy), "accessible" (7 colors, Okabe-Ito inspired). Sequential: "maroon_seq", "orange_seq", "navy_seq". Diverging: "diverging". Neutral: "gray_seq". Default: "main".
n
Number of colors to return. NULL returns all base palette colors. Sequential and diverging palettes interpolate when n exceeds the base size.
reverse
Reverse the palette order? Default: FALSE.
alpha
Opacity, 0–1. Default: 1.

Value

A character vector of hex color codes.

Examples

vvn_palette()                        # 10 main colors
vvn_palette("brand")                 # "#861F41" "#E5751F" "#1B5299"
vvn_palette("maroon_seq", n = 9)     # 9-step interpolated maroon ramp
vvn_palette("diverging", n = 11)
vvn_palette("accessible")

view_vvn_palette

Display a horizontal color swatch strip for any VVN palette. Returns a ggplot object that can be saved with vvn_save().

Usage

view_vvn_palette(palette = "main", n = NULL, label = TRUE, title = NULL)

Arguments

palette
Palette name (see vvn_palette()). Default: "main".
n
Number of colors to display. NULL shows all base palette colors.
label
Show hex codes on each swatch? Default: TRUE.
title
Plot title. Defaults to vvn_palette("<palette>").

Value

A ggplot object, printed invisibly.

Examples

view_vvn_palette()                       # "main" palette
view_vvn_palette("diverging", n = 9)
view_vvn_palette("accessible", label = FALSE)
vvn_save(view_vvn_palette("brand"), "figures/brand_palette.png")

Scales

scale_color_vvn

Apply a VVN categorical color palette to the color aesthetic of a ggplot2 chart. scale_colour_vvn() is a British-English alias.

Usage

scale_color_vvn(palette = "main", reverse = FALSE, alpha = 1, ...)
scale_colour_vvn(palette = "main", reverse = FALSE, alpha = 1, ...)

Arguments

palette
Palette name: "main" (general purpose, up to 10 groups), "brand" (3 groups: maroon / orange / navy), or "accessible" (colorblind-safe). Default: "main".
reverse
Reverse palette order? Default: FALSE.
alpha
Opacity, 0–1. Default: 1.
...
Passed to ggplot2::discrete_scale().

Value

A ggplot2 scale object.

Examples

library(ggplot2)

ggplot(economics_long, aes(date, value01, color = variable)) +
  geom_line() +
  scale_color_vvn("main")

ggplot(df, aes(date, value, color = group)) +
  geom_line() +
  scale_color_vvn("brand")   # 2–3 groups

scale_fill_vvn

Apply a VVN categorical color palette to the fill aesthetic. Same palettes and arguments as scale_color_vvn().

Usage

scale_fill_vvn(palette = "main", reverse = FALSE, alpha = 1, ...)

Arguments

palette
Palette name: "main", "brand", or "accessible". Default: "main".
reverse
Reverse palette order? Default: FALSE.
alpha
Opacity, 0–1. Default: 1.
...
Passed to ggplot2::discrete_scale().

Value

A ggplot2 scale object.

Examples

library(ggplot2)

# Grouped bar chart
ggplot(mpg, aes(class, fill = drv)) +
  geom_bar(position = "dodge") +
  scale_fill_vvn("brand")

# Colorblind-safe palette
ggplot(df, aes(region, fill = category)) +
  geom_col() +
  scale_fill_vvn("accessible")

scale_color_vvn_c

Apply a VVN sequential or diverging palette to a continuous color aesthetic. scale_colour_vvn_c() is a British-English alias.

Usage

scale_color_vvn_c(palette = "maroon_seq", reverse = FALSE, ...)
scale_colour_vvn_c(palette = "maroon_seq", reverse = FALSE, ...)

Arguments

palette
Palette name: "maroon_seq" (primary), "orange_seq", "navy_seq", "diverging" (for above/below-average data), or "gray_seq". Default: "maroon_seq".
reverse
Reverse palette direction? Default: FALSE.
...
Passed to ggplot2::scale_color_gradientn().

Value

A ggplot2 scale object.

Examples

ggplot(df, aes(x, y, color = value)) +
  geom_point(size = 3) +
  scale_color_vvn_c("maroon_seq")

ggplot(df, aes(x, y, color = z_score)) +
  geom_point() +
  scale_color_vvn_c("diverging")

scale_fill_vvn_c

Apply a VVN sequential or diverging palette to a continuous fill aesthetic. The primary function for county choropleth maps.

Usage

scale_fill_vvn_c(palette = "maroon_seq", reverse = FALSE, ...)

Arguments

palette
Palette name: "maroon_seq" (default), "orange_seq", "navy_seq", "diverging", or "gray_seq".
reverse
Reverse palette direction? Default: FALSE.
...
Passed to ggplot2::scale_fill_gradientn().

Value

A ggplot2 scale object.

Examples

library(ggplot2); library(sf)

# Standard county choropleth
ggplot(va_counties_sf, aes(fill = pct_broadband)) +
  geom_sf(color = "#FFFFFF", linewidth = 0.3) +
  scale_fill_vvn_c("maroon_seq") +
  theme_vvn_map()

# Diverging scale (above/below state average)
ggplot(va_counties_sf, aes(fill = dev_from_mean)) +
  geom_sf() +
  scale_fill_vvn_c("diverging")

Chart Helpers

vvn_title

Add a formatted title and optional subtitle to a ggplot.

Usage

vvn_title(title, subtitle = NULL)

Arguments

title
Title string. Use headline case (capitalize all major words).
subtitle
Optional subtitle. Use sentence case. Default: NULL.

Value

A ggplot2::labs() call.

Examples

ggplot(mpg, aes(displ, hwy)) +
  geom_point() +
  vvn_title("Engine Size vs Fuel Economy",
             subtitle = "Motor Trend car road tests, 1974")

vvn_source

Add a source attribution caption below a ggplot. Required on every published VVN figure.

Usage

vvn_source(source, note = NULL)

Arguments

source
Source string, e.g. "U.S. Census Bureau, ACS 5-Year Estimates, 2022".
note
Optional methodological note. Prepended as Note: … before the source text. Default: NULL.

Value

A ggplot2::labs(caption = ...) call.

Examples

ggplot(df, aes(x, y)) +
  geom_col() +
  vvn_source("U.S. Census Bureau, ACS 5-Year Estimates, 2022")

# With a methodological note
ggplot(df, aes(x, y)) +
  geom_col() +
  vvn_source("Virginia Employment Commission",
              note = "Data suppressed for localities with fewer than 100 workers.")

vvn_note

Add a footnote-style annotation below a ggplot (without a source line).

Usage

vvn_note(text)

Arguments

text
Note text. Prefixed with Note: in the caption.

Value

A ggplot2::labs(caption = ...) call.

Examples

ggplot(df, aes(x, y)) +
  geom_col() +
  vvn_note("Excludes localities with suppressed data.")

remove_ticks

Remove axis tick marks from a ggplot. Recommended for bar charts and categorical axes.

Usage

remove_ticks(axis = "both")

Arguments

axis
Which axis ticks to remove: "both" (default), "x", or "y".

Value

A ggplot2::theme() call.

Examples

ggplot(df, aes(x = value, y = region)) +
  geom_col() +
  remove_ticks()

remove_axis

Remove an entire axis — text, title, ticks, and line — from a ggplot.

Usage

remove_axis(axis = "x")

Arguments

axis
Which axis to remove entirely: "x" (default), "y", or "both".

Value

A ggplot2::theme() call.

Examples

  geom_col() +
  remove_axis("x")   # when category labels are self-evident

scatter_grid

Add major grid lines on both x and y axes. Use with theme_vvn() for scatter plots where both axes need a visual reference.

Usage

scatter_grid()

Value

A ggplot2::theme() call.

ggplot(df, aes(income, broadband)) +
  geom_point() +

Remove the legend from a ggplot entirely.

remove_legend()

vvn_save

Save a ggplot at publication quality with VVN default dimensions (8 × 5 inches, 300 dpi, white background). Supports multi-format export in a single call.

Usage

vvn_save(plot = ggplot2::last_plot(), filename, width = 8, height = 5,
         dpi = 300, formats = NULL)

Arguments

plot
A ggplot object. Defaults to ggplot2::last_plot().
filename
File path with extension (e.g. "figures/chart.png"), or base path without extension when formats is provided.
width
Output width in inches. Default: 8.
height
Output height in inches. Default: 5.
dpi
Resolution in dots per inch. Default: 300.
formats
Character vector of file extensions for simultaneous multi-format export, e.g. c("png", "pdf", "svg"). When provided, overrides the extension in filename.

Value

Invisibly returns the file path(s) written.

Examples

p <- ggplot(mpg, aes(class)) + geom_bar() + theme_vvn()

vvn_save(p, "figures/class_bar.png")

# Custom size
vvn_save(p, "figures/class_bar_wide.png", width = 12, height = 4)

# Export to multiple formats at once
vvn_save(p, "figures/class_bar", formats = c("png", "pdf", "svg"))

vvn_accessibility_check

Compute WCAG 2.1 contrast ratios for a set of hex colors against a background color and report pass/fail status.

Usage

vvn_accessibility_check(colors, background = "#FFFFFF", level = c("AA", "AAA"))

Arguments

colors
Character vector of hex codes to test.
background
Background hex code to test against. Default: "#FFFFFF".
level
WCAG standard: "AA" requires ≥ 4.5:1 contrast ratio (default); "AAA" requires ≥ 7.0:1.

Value

Invisibly returns a data frame with columns color, contrast_ratio, pass. Prints a formatted pass/fail report via cli.

Examples

# Check brand palette on white background
vvn_accessibility_check(vvn_palette("brand"))

# Check white text on maroon background
vvn_accessibility_check(c("#FFFFFF", "#F7F7F7"), background = "#861F41")

# Stricter AAA standard
vvn_accessibility_check(vvn_palette("accessible"), level = "AAA")

Tables

vvn_table

Apply VVN branding to a gt table: VT Maroon column headers, alternating light gray rows, orange accent border on the header, and optional title and source note.

Requires the gt package (install.packages("gt")).

Usage

vvn_table(gt_tbl, title = NULL, subtitle = NULL, source_note = NULL,
          striped = TRUE, compact = FALSE)

Arguments

gt_tbl
A gt_tbl object created by gt::gt().
title
Table title string. Use headline case. Default: NULL.
subtitle
Optional subtitle. Use sentence case. Default: NULL.
source_note
Source attribution string. Required for all published tables. Default: NULL.
striped
Apply alternating row shading (#F7F7F7)? Default: TRUE.
compact
Reduce cell padding for dense tables? Default: FALSE.

Value

A styled gt_tbl object.

Examples

library(gt)

gt(head(mtcars)) |>
  vvn_table(
    title       = "Motor Trend Road Tests",
    subtitle    = "Selected vehicles, 1974",
    source_note = "Source: Henderson & Velleman (1981)"
  )

vvn_table_pvalues

Color-code a p-value column in a gt regression table: maroon for p < 0.001, orange for p < 0.05. Pipe after vvn_table().

Usage

vvn_table_pvalues(gt_tbl, col_name = "p.value",
                  thresholds = c("0.001" = "#861F41", "0.05" = "#E5751F"))

Arguments

gt_tbl
A gt_tbl object, typically piped from vvn_table().
col_name
Name of the column containing p-values. Default: "p.value".
thresholds
Named character vector mapping threshold values to highlight hex colors. Default highlights p < 0.001 in maroon and p < 0.05 in orange.

Value

A gt_tbl with color-highlighted p-value cells.

Examples

library(gt); library(broom)

lm(mpg ~ wt + cyl, data = mtcars) |>
  tidy() |>
  gt() |>
  vvn_table(title = "OLS Regression: Fuel Economy") |>
  vvn_table_pvalues()

Maps

vvn_map_style

Apply VVN branding to a Leaflet choropleth map: muted CartoDB basemap, VVN palette fill, maroon county hover highlight, popup tooltips, and a styled legend.

Requires the leaflet and htmltools packages.

Usage

vvn_map_style(map, data, value_col, label_col = value_col,
              palette = "maroon_seq", n_bins = 6, title = value_col,
              opacity = 0.75,
              fmt_fn = function(x) format(round(x), big.mark = ","))

Arguments

map
A leaflet map object from leaflet::leaflet().
data
An sf data frame with polygon geometry in CRS 4326.
value_col
Name of the numeric column to visualize (character). Use a normalized variable — rate, percentage, or per-capita — never a raw count.
label_col
Column used as the tooltip label (e.g. county name). Default: same as value_col.
palette
VVN palette: "maroon_seq" (default), "orange_seq", "navy_seq", or "diverging".
n_bins
Number of legend bins. Default: 6.
title
Legend title string. Default: value_col.
opacity
Fill opacity, 0–1. Default: 0.75.
fmt_fn
Function to format tooltip values. Default: format(round(x), big.mark = ",").

Value

A styled leaflet map.

Examples

library(leaflet); library(sf)

leaflet(va_counties) |>
  vvn_map_style(va_counties, "pct_broadband",
                label_col = "NAME",
                title     = "Broadband coverage (%)",
                palette   = "maroon_seq")

vvn_button

A styled Shiny action button in one of three VVN style variants.

vvn_button(inputId, label, icon = NULL,

Arguments

inputId
Shiny input ID.

Button label text.

Optional icon from shiny::icon(). Default: NULL.

vvn_kpi_card

Render a VVN KPI metric card with a large value, descriptive label, and optional trend indicator. Use three per dashboard row — one maroon, one orange, one navy.

Usage

vvn_kpi_card(value, label, trend = NULL, trend_text = NULL,
             color_scheme = c("maroon", "orange", "navy"))

Arguments

value
Scalar metric value displayed in large type, e.g. "76%" or "$52,342".
label
Short descriptive label below the value, e.g. "Households with broadband".
trend
Trend direction: "up", "down", or NULL for no indicator. Default: NULL.
trend_text
Trend annotation string, e.g. "+5 pp vs 2020". Default: NULL.
color_scheme
Accent color for the top border and trend icon: "maroon" (default), "orange", or "navy".

Value

A Shiny div tag.

Examples

library(shiny)

fluidRow(
  column(4, vvn_kpi_card("76%",     "Households with broadband",
                          trend = "up", trend_text = "+5 pp vs 2020")),
  column(4, vvn_kpi_card("$52,342", "Median household income",
                          trend = "up", trend_text = "+8% vs 2019",
                          color_scheme = "orange")),
  column(4, vvn_kpi_card("22.4%",   "Population in poverty",
                          trend = "down", trend_text = "-2.1 pp vs 2020",
                          color_scheme = "navy"))
)

Scaffolding

create_vvn_dashboard

Scaffold a production-ready R Shiny dashboard with VVN branding: maroon navbar, KPI cards, reactive charts, Leaflet county map, sidebar filters, and a data download tab. The generated project runs immediately with placeholder data.

Usage

create_vvn_dashboard(name, path = ".", title = name,
                     author = "Visualizing Virginia's Numbers")

Arguments

name
Project folder name (character). Use snake_case or kebab-case, e.g. "housing_dashboard".
path
Parent directory. Default: "." (current working directory).
title
Dashboard title displayed in the maroon navbar. Default: name.
author
Author name(s) injected into the README. Default: "Visualizing Virginia's Numbers".

Value

Invisibly returns the project path (character). Prints setup instructions via cli.

Details

Creates the following structure:

name/
├── app.R            ← main dashboard (runs immediately)
├── R/data_prep.R    ← load and clean your data here
├── www/vvn.css      ← VVN brand CSS
└── data/
    ├── raw/
    └── processed/

Examples

create_vvn_dashboard("housing_dashboard",
                      title = "Virginia Housing Affordability Dashboard")

shiny::runApp("housing_dashboard")       # test locally
rsconnect::deployApp("housing_dashboard")   # publish

                 author = "Visualizing Virginia's Numbers")

Arguments

## Tables

### vvn_table {#vvn_table}

Apply VVN branding to a `gt` table: VT Maroon column headers, alternating light gray rows, orange accent border on the header, and optional title and source note.

Requires the `gt` package (`install.packages("gt")`).

#### Usage

```r
vvn_table(gt_tbl, title = NULL, subtitle = NULL, source_note = NULL,
          striped = TRUE, compact = FALSE)
```

#### Arguments

```{=html}
<dl class="refman-dl">
<dt>gt_tbl</dt>
<dd>A <code>gt_tbl</code> object created by <code>gt::gt()</code>.</dd>
<dt>title</dt>
<dd>Table title string. Use <strong>headline case</strong>. Default: <code>NULL</code>.</dd>
<dt>subtitle</dt>
<dd>Optional subtitle. Use <strong>sentence case</strong>. Default: <code>NULL</code>.</dd>
<dt>source_note</dt>
<dd>Source attribution string. Required for all published tables. Default: <code>NULL</code>.</dd>
<dt>striped</dt>
<dd>Apply alternating row shading (<code>#F7F7F7</code>)? Default: <code>TRUE</code>.</dd>
<dt>compact</dt>
<dd>Reduce cell padding for dense tables? Default: <code>FALSE</code>.</dd>
</dl>
```

#### Value

A styled `gt_tbl` object.

#### Examples

```r
library(gt)

gt(head(mtcars)) |>
  vvn_table(
    title       = "Motor Trend Road Tests",
    subtitle    = "Selected vehicles, 1974",
    source_note = "Source: Henderson & Velleman (1981)"
  )
```

### vvn_table_pvalues {#vvn_table_pvalues}

Color-code a p-value column in a `gt` regression table: maroon for `p < 0.001`, orange for `p < 0.05`. Pipe after `vvn_table()`.

#### Usage

```r
vvn_table_pvalues(gt_tbl, col_name = "p.value",
                  thresholds = c("0.001" = "#861F41", "0.05" = "#E5751F"))
```

#### Arguments

```{=html}
<dl class="refman-dl">
<dt>gt_tbl</dt>
<dd>A <code>gt_tbl</code> object, typically piped from <code>vvn_table()</code>.</dd>
<dt>col_name</dt>
<dd>Name of the column containing p-values. Default: <code>"p.value"</code>.</dd>
<dt>thresholds</dt>
<dd>Named character vector mapping threshold values to highlight hex colors. Default highlights <code>p &lt; 0.001</code> in maroon and <code>p &lt; 0.05</code> in orange.</dd>
</dl>
```

#### Value

A `gt_tbl` with color-highlighted p-value cells.

#### Examples

```r
library(gt); library(broom)

lm(mpg ~ wt + cyl, data = mtcars) |>
  tidy() |>
  gt() |>
  vvn_table(title = "OLS Regression: Fuel Economy") |>
  vvn_table_pvalues()
```

## Maps

### vvn_map_style {#vvn_map_style}

Apply VVN branding to a Leaflet choropleth map: muted CartoDB basemap, VVN palette fill, maroon county hover highlight, popup tooltips, and a styled legend.

Requires the `leaflet` and `htmltools` packages.

#### Usage

```r
vvn_map_style(map, data, value_col, label_col = value_col,
              palette = "maroon_seq", n_bins = 6, title = value_col,
              opacity = 0.75,
              fmt_fn = function(x) format(round(x), big.mark = ","))
```

#### Arguments

```{=html}
<dl class="refman-dl">
<dt>map</dt>
<dd>A <code>leaflet</code> map object from <code>leaflet::leaflet()</code>.</dd>
<dt>data</dt>
<dd>An <code>sf</code> data frame with polygon geometry in CRS 4326.</dd>
<dt>value_col</dt>
<dd>Name of the numeric column to visualize (character). Use a normalized variable — rate, percentage, or per-capita — never a raw count.</dd>
<dt>label_col</dt>
<dd>Column used as the tooltip label (e.g. county name). Default: same as <code>value_col</code>.</dd>
<dt>palette</dt>
<dd>VVN palette: <code>"maroon_seq"</code> (default), <code>"orange_seq"</code>, <code>"navy_seq"</code>, or <code>"diverging"</code>.</dd>
<dt>n_bins</dt>
<dd>Number of legend bins. Default: <code>6</code>.</dd>
<dt>title</dt>
<dd>Legend title string. Default: <code>value_col</code>.</dd>
<dt>opacity</dt>
<dd>Fill opacity, 0–1. Default: <code>0.75</code>.</dd>
<dt>fmt_fn</dt>
<dd>Function to format tooltip values. Default: <code>format(round(x), big.mark = ",")</code>.</dd>
</dl>
```

#### Value

A styled `leaflet` map.

#### Examples

```r
library(leaflet); library(sf)

leaflet(va_counties) |>
  vvn_map_style(va_counties, "pct_broadband",
                label_col = "NAME",
                title     = "Broadband coverage (%)",
                palette   = "maroon_seq")
```

## Shiny UI

### vvn_filter {#vvn_filter}

A VVN-branded Shiny dropdown filter (`selectInput`) in a styled container with a maroon label.

Requires the `shiny` package.

#### Usage

```r
vvn_filter(inputId, label, choices, selected = NULL,
           multiple = FALSE, width = "100%")
```

#### Arguments

```{=html}
<dl class="refman-dl">
<dt>inputId</dt>
<dd>Shiny input ID (character).</dd>
<dt>label</dt>
<dd>Display label shown above the dropdown.</dd>
<dt>choices</dt>
<dd>Named vector or list of choices passed to <code>shiny::selectInput()</code>.</dd>
<dt>selected</dt>
<dd>Default selected value(s). Default: <code>NULL</code> (first choice).</dd>
<dt>multiple</dt>
<dd>Allow multiple selections? Default: <code>FALSE</code>.</dd>
<dt>width</dt>
<dd>CSS width of the filter container. Default: <code>"100%"</code>.</dd>
</dl>
```

#### Value

A Shiny `tagList`.

#### Examples

```r
vvn_filter("year_sel", "Select year",
           choices  = 2018:2023,
           selected = 2023)

vvn_filter("region_sel", "Region",
           choices  = c("All", "Rural", "Urban"),
           selected = "All")
```

### vvn_button {#vvn_button}

A styled Shiny action button in one of three VVN style variants.

#### Usage

```r
vvn_button(inputId, label, icon = NULL,
           style = c("primary", "secondary", "outline"), width = NULL)
```

#### Arguments

```{=html}
<dl class="refman-dl">
<dt>inputId</dt>
<dd>Shiny input ID.</dd>
<dt>label</dt>
<dd>Button label text.</dd>
<dt>icon</dt>
<dd>Optional icon from <code>shiny::icon()</code>. Default: <code>NULL</code>.</dd>
<dt>style</dt>
<dd><code>"primary"</code> — maroon fill, white text (use for Apply Filters). <code>"secondary"</code> — orange fill, white text. <code>"outline"</code> — maroon border, white background (use for Reset). Default: <code>"primary"</code>.</dd>
<dt>width</dt>
<dd>CSS width. Default: <code>NULL</code> (auto).</dd>
</dl>
```

#### Value

A Shiny `actionButton`.

#### Examples

```r
vvn_button("apply_btn",  "Apply Filters", style = "primary")
vvn_button("reset_btn",  "Reset",         style = "outline")
vvn_button("export_btn", "Download CSV",  style = "secondary",
           icon = shiny::icon("download"))
```

### vvn_kpi_card {#vvn_kpi_card}

Render a VVN KPI metric card with a large value, descriptive label, and optional trend indicator. Use three per dashboard row — one maroon, one orange, one navy.

#### Usage

```r
vvn_kpi_card(value, label, trend = NULL, trend_text = NULL,
             color_scheme = c("maroon", "orange", "navy"))
```

#### Arguments

```{=html}
<dl class="refman-dl">
<dt>value</dt>
<dd>Scalar metric value displayed in large type, e.g. <code>"76%"</code> or <code>"$52,342"</code>.</dd>
<dt>label</dt>
<dd>Short descriptive label below the value, e.g. <code>"Households with broadband"</code>.</dd>
<dt>trend</dt>
<dd>Trend direction: <code>"up"</code>, <code>"down"</code>, or <code>NULL</code> for no indicator. Default: <code>NULL</code>.</dd>
<dt>trend_text</dt>
<dd>Trend annotation string, e.g. <code>"+5 pp vs 2020"</code>. Default: <code>NULL</code>.</dd>
<dt>color_scheme</dt>
<dd>Accent color for the top border and trend icon: <code>"maroon"</code> (default), <code>"orange"</code>, or <code>"navy"</code>.</dd>
</dl>
```

#### Value

A Shiny `div` tag.

#### Examples

```r
library(shiny)

fluidRow(
  column(4, vvn_kpi_card("76%",     "Households with broadband",
                          trend = "up", trend_text = "+5 pp vs 2020")),
  column(4, vvn_kpi_card("$52,342", "Median household income",
                          trend = "up", trend_text = "+8% vs 2019",
                          color_scheme = "orange")),
  column(4, vvn_kpi_card("22.4%",   "Population in poverty",
                          trend = "down", trend_text = "-2.1 pp vs 2020",
                          color_scheme = "navy"))
)
```

## Scaffolding

### create_vvn_dashboard {#create_vvn_dashboard}

Scaffold a production-ready R Shiny dashboard with VVN branding: maroon navbar, KPI cards, reactive charts, Leaflet county map, sidebar filters, and a data download tab. The generated project runs immediately with placeholder data.

#### Usage

```r
create_vvn_dashboard(name, path = ".", title = name,
                     author = "Visualizing Virginia's Numbers")
```

#### Arguments

```{=html}
<dl class="refman-dl">
<dt>name</dt>
<dd>Project folder name (character). Use snake_case or kebab-case, e.g. <code>"housing_dashboard"</code>.</dd>
<dt>path</dt>
<dd>Parent directory. Default: <code>"."</code> (current working directory).</dd>
<dt>title</dt>
<dd>Dashboard title displayed in the maroon navbar. Default: <code>name</code>.</dd>
<dt>author</dt>
<dd>Author name(s) injected into the README. Default: <code>"Visualizing Virginia's Numbers"</code>.</dd>
</dl>
```

#### Value

Invisibly returns the project path (character). Prints setup instructions via `cli`.

#### Details

Creates the following structure:

```
name/
├── app.R            ← main dashboard (runs immediately)
├── R/data_prep.R    ← load and clean your data here
├── www/vvn.css      ← VVN brand CSS
└── data/
    ├── raw/
    └── processed/
```

#### Examples

```r
create_vvn_dashboard("housing_dashboard",
                      title = "Virginia Housing Affordability Dashboard")

shiny::runApp("housing_dashboard")       # test locally
rsconnect::deployApp("housing_dashboard")   # publish
```

### create_vvn_story {#create_vvn_story}

Scaffold a new VVN Insights Quarto project with VVN branding, directory structure, and pre-built narrative sections (Overview, Background, Findings, Conclusion). The generated document renders immediately with placeholder content.

#### Usage

```r
create_vvn_story(name, path = ".", title = name,
                 author = "Visualizing Virginia's Numbers")
```

#### Arguments

```{=html}
<dl class="refman-dl">
<dt>name</dt>
<dd>Project folder name. Use snake_case, e.g. <code>"childcare_cost"</code>.</dd>
<dt>path</dt>
<dd>Parent directory. Default: <code>"."</code>.</dd>
<dt>title</dt>
<dd>Story title injected into the YAML front matter and hero banner. Default: <code>name</code>.</dd>
<dt>author</dt>
<dd>Author name(s). Default: <code>"Visualizing Virginia's Numbers"</code>.</dd>
</dl>
```

#### Value

Invisibly returns the project path. Prints setup instructions via `cli`.

#### Details

Creates the following structure:

```
name/
├── index.qmd        ← edit this — pre-built narrative sections
├── _quarto.yml
├── styles.scss      ← VVN brand SCSS
├── data/raw/
├── data/processed/
├── figures/
└── scripts/
```

#### Examples

```r
create_vvn_story("childcare_cost",
                  title  = "Childcare Cost in Rural Virginia",
                  author = "VVN Research Team")

quarto::quarto_render("childcare_cost/index.qmd")
```

### check_vvn_story {#check_vvn_story}

Validate a VVN Insights `index.qmd` for required sections and VVN function usage before publishing. Checks for `Overview`, `Background`, `Findings`, `Conclusion`, `theme_vvn()`, and `vvn_source()`.

#### Usage

```r
check_vvn_story(path = ".")
```

#### Arguments

```{=html}
<dl class="refman-dl">
<dt>path</dt>
<dd>Path to the story folder containing <code>index.qmd</code>. Default: <code>"."</code>.</dd>
</dl>
```

#### Value

Invisibly `TRUE` if all checks pass, `FALSE` otherwise. Prints a formatted pass/fail report.

#### Examples

```r
create_vvn_story("my_insight")
# ... edit index.qmd ...
check_vvn_story("my_insight")
```
