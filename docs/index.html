About vvnthemes

vvnthemes is the design system for all VVN projects. It provides Virginia Tech brand colors, ggplot2 themes, map styling, styled tables, Shiny UI components, and project scaffolding — so every chart, map, table, dashboard, and narrative report your team produces looks consistent and publication-ready.

🎨
VT Brand Colors
VT Maroon, Burnt Orange, and Navy applied globally with a single call to set_vvn_defaults(). Discrete and continuous palettes for every chart type.
📊
ggplot2 Themes & Scales
Three themes for charts, maps, and minimal layouts. Categorical and continuous color scales with automatic VT palette mapping.
🗺️
Interactive Maps
One-function Leaflet county choropleth maps with VT branded fill, maroon hover highlight, tooltips, and a styled legend.
📋
Styled Tables
Branded gt tables with maroon headers, alternating rows, source notes, and p-value color coding for regression output.
🖥️
Shiny UI Components
Branded filters, sliders, buttons, and KPI cards for Shiny dashboards — all styled to match VT brand guidelines.
🚀
Project Scaffolding
One-command setup for a complete Shiny dashboard or Quarto narrative report. Runs immediately with placeholder data.

Installation

Recommended — no extra packages needed in R 4.4+:

pak::pkg_install("yujuangao/VVN")

Alternatives:

# Option B
install.packages("remotes")
remotes::install_github("yujuangao/VVN")

# Option C
install.packages("devtools")
devtools::install_github("yujuangao/VVN")

After installation, load the package and apply VT brand defaults for the session:

library(vvnthemes)
set_vvn_defaults()   # sets theme_vvn() + VT Maroon geom colors globally

Dependencies installed automatically: ggplot2, scales, rlang, dplyr

Optional packages (install separately if needed): gt (tables), leaflet + sf (maps), shiny + bslib + bsicons (dashboards), broom (regression tables)

NoteNamespace conflict with urbnthemes

If urbnthemes is also loaded, its remove_legend(), remove_ticks(), and scatter_grid() will mask the vvnthemes versions. Use the vvn_-prefixed aliases to be explicit:

vvn_remove_legend()   # always resolves to vvnthemes
vvn_remove_ticks()
vvn_remove_axis()
vvn_legend_bottom()
vvn_scatter_grid()
WarningDeveloper install (working on the package source)

Always use R CMD build + R CMD INSTALL — not devtools::install() — to avoid a corrupt help database error (lazy-load database … is corrupt):

cd /path/to/VVN
R CMD build vvnthemes --no-build-vignettes
R CMD INSTALL vvnthemes_0.0.1.tar.gz
rm vvnthemes_0.0.1.tar.gz

vvnthemes is organized into 9 modules. Each module is independent — use only what your project needs.

Typical Workflow

For most VVN projects, follow this five-step pattern:

1
Install & load
Install from GitHub once. At the top of every script, call library(vvnthemes) and set_vvn_defaults() — this applies the VT theme and colors to all subsequent charts automatically.
2
Load and prepare data
Import your county-level data. VVN maps require a normalized variable — a rate, percentage, or per-capita figure, never a raw count. Join to va_counties by FIPS code.
3
Build the visualization
Use ggplot2 + theme_vvn() for charts, vvn_map_style() for interactive maps, or gt() |> vvn_table() for tables. Add VVN scales to apply the correct color palette.
4
Add titles and source attribution
Every published VVN figure requires vvn_title() for the headline (title case) and vvn_source() for the data citation. These are non-negotiable requirements before any figure is shared.
5
Save or deploy
Export static figures at 300 dpi with vvn_save(). For dashboards, deploy to shinyapps.io with rsconnect::deployApp(). For narrative reports, render with quarto render.

## Demo: Build a Choropleth Map

Load `vvnthemes`, join your county data, and call `vvn_map_style()` — the map is styled, labeled, and ready in under 10 lines of code.

```{=html}
<div class="demo-panel">
<div class="demo-panel-code">
<div class="demo-panel-header">vvnthemes · choropleth_map.R</div>
<div class="demo-panel-body">
<pre><code id="demo-gs-code"></code><span class="demo-cursor" id="demo-gs-cursor"></span></pre>
</div>
</div>
<div class="demo-panel-output" id="demo-gs-output" style="opacity:0;transition:opacity 0.7s ease;">
<div class="demo-panel-header">Output</div>
<div class="demo-panel-body">
<img src="images/demo-map.png" alt="VVN county choropleth map of Virginia">
</div>
</div>
</div>
<script>
(function(){
var code = [
'library(vvnthemes)',
'library(ggplot2)',
'library(sf)',
'',
'# Load Virginia county boundaries',
'va <- vvnthemes::va_counties',
'',
'# Join your data (FIPS + normalized value)',
'va$broadband <- your_county_data$rate',
'',
'# Build choropleth map',
'ggplot(va) +',
'  geom_sf(aes(fill = broadband), color = "white") +',
'  scale_fill_vvn_c("maroon_seq",',
'    name   = "Broadband Access",',
'    labels = scales::label_percent()) +',
'  theme_vvn_map() +',
'  vvn_title("Broadband Access by County, 2023") +',
'  vvn_source("American Community Survey")'
].join('\n');
function startDemo() {
  var el = document.getElementById('demo-gs-code');
  var cur = document.getElementById('demo-gs-cursor');
  var out = document.getElementById('demo-gs-output');
  if (!el) return;
  var i = 0, revealed = false;
  function type(){
    if(i < code.length){
      el.textContent += code[i++];
      if(!revealed && i >= Math.floor(code.length * 0.2)){
        revealed = true;
        if(out) out.style.opacity = '1';
      }
      setTimeout(type, i < 80 ? 35 : 16);
    } else {
      if(cur) cur.style.display = 'none';
      if(!revealed && out) out.style.opacity = '1';
    }
  }
  setTimeout(type, 700);
}
if(document.readyState === 'loading'){
  document.addEventListener('DOMContentLoaded', startDemo);
} else {
  startDemo();
}
})();
</script>
```
