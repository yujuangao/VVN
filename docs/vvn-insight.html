VVN Insight

Scaffold and build a complete VVN narrative report.

The create_vvn_story() function scaffolds a complete Quarto narrative report in one command. The analysis and the narrative are kept in separate files so each is easier to work on independently.

Create the project

library(vvnthemes)
create_vvn_story(
  "my-insight",
  title  = "Broadband Access in Rural Virginia",
  author = "VVN Research Team"
)

Project structure

my-insight/
├── index.qmd            # Narrative document — fill in text, insert figure names
├── _quarto.yml          # Quarto configuration
├── styles.scss          # VVN brand styles (do not edit)
├── data/
│   ├── raw/             # Place raw data files here
│   └── processed/       # Place cleaned / analysis-ready data here
├── scripts/
│   └── analysis.R       # Build all figures here, then run this script
└── figures/             # Auto-created when analysis.R runs — do not edit manually

What each file does

scripts/analysis.R is where you write all chart code. It loads your data, builds each figure using VVN functions, and saves numbered PNGs to figures/ automatically. Run this script first, before opening index.qmd.

index.qmd is the narrative document. It contains all section placeholders ready to fill in. Figures are embedded using a file.exists() guard — the document renders cleanly at any stage, even before figures are built. No chart code lives here.

figures/ is populated automatically when you run scripts/analysis.R. Files are named 01_, 02_, … in the order the script runs them.

data/raw/ holds your original data files unchanged. data/processed/ holds cleaned, analysis-ready files. Load processed files in scripts/analysis.R.

_quarto.yml configures the Quarto project: theme, TOC settings, figure dimensions, and output format. You rarely need to edit this file.

styles.scss is the compiled VVN brand stylesheet. Do not edit this file.

Document structure

Section Element Required
Hero banner ::: {.hero} div Yes
Overview and key finding > blockquote Yes
Background ## section Yes
Data and Methods ## section + table Yes
Findings ## section + charts + map + table Yes
Discussion ## section Yes
Conclusion ## section Yes

For section requirements, narrative rules, and publishing checklist, see VVN Insights Standards →

Workflow

Follow these three steps in order:

Step 1 — Build figures

Open scripts/analysis.R in RStudio and fill in your chart code. The script contains a gallery of common VVN chart types — uncomment the ones you need and fill in your column names and labels. Then source the script:

# In RStudio: open scripts/analysis.R, then click Source
# Or from the R console:
source("scripts/analysis.R")

Figures are auto-numbered (01_, 02_, …) and saved to figures/ in the order they run.

Step 2 — Write the narrative

Open index.qmd. Replace every [placeholder] with your content. For each figure chunk, update the filename inside f to match the file saved in figures/:

# In index.qmd — replace [name] with your actual figure filename:
f <- "figures/01_trend_statewide.png"
if (file.exists(f)) knitr::include_graphics(f)

The file.exists() guard means index.qmd renders without errors even if some figures are not yet built — missing figures simply leave a blank space. Once the correct filename is set and the figure exists, it appears automatically.

Step 3 — Render

quarto::quarto_render("index.qmd")

In RStudio, press Cmd+Shift+K (Ctrl+Shift+K on Windows).

## Publishing checklist

Before submitting a VVN Insight for publication:

- [ ] `scripts/analysis.R` runs without errors and all figures appear in `figures/`
- [ ] All `[...]` placeholders in `index.qmd` replaced
- [ ] Figure filenames inside `f <- "figures/..."` in each chunk match actual files in `figures/`
- [ ] Hero banner present with title and VVN subtitle
- [ ] Key finding stated clearly in Overview
- [ ] Background explains context and audience
- [ ] Data & Methods table complete with source
- [ ] At least 2 charts with titles, subtitles, and source lines
- [ ] At least 1 map (interactive or static)
- [ ] Summary table with source note
- [ ] Discussion covers interpretation and limitations
- [ ] Conclusion with contact information
- [ ] All chart titles use headline case
- [ ] All chart subtitles use sentence case
- [ ] `embed-resources: true` in YAML (single-file HTML)
