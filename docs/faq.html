FAQ

Common questions about vvnthemes and VVN products.

Installation and setup

What R packages do I need?

The core package is vvnthemes. Additional packages are required for specific features:

Feature Required packages
Charts vvnthemes, ggplot2
Static maps vvnthemes, ggplot2, sf, tigris
Interactive maps vvnthemes, leaflet, sf
Tables vvnthemes, gt
Shiny dashboards vvnthemes, shiny, bslib
Quarto Insights vvnthemes, ggplot2, gt

Install everything at once:

install.packages(c("ggplot2", "sf", "tigris", "leaflet", "gt", "shiny", "bslib"))
devtools::install_github("vt-vvn/vvnthemes")

What is the difference between theme_vvn() and set_vvn_defaults()?

theme_vvn() is a ggplot2 theme — add it to individual charts with +:

ggplot(df, aes(x, y)) + geom_col() + theme_vvn()

set_vvn_defaults() sets theme_vvn() as the global default so you never have to add it explicitly. It also updates the default colors for all geoms (bars, lines, points → VT Maroon). Call it once at the top of your script:

library(vvnthemes)
set_vvn_defaults()   # now every chart automatically uses theme_vvn()

Use set_vvn_defaults() in scripts and Quarto documents. Use theme_vvn() directly only when you need to mix VVN and non-VVN charts in the same session.

Can I use vvnthemes outside of Virginia?

Yes. The package is a general-purpose ggplot2 theming system built on Virginia Tech’s brand. The design system works for any data. The only Virginia-specific pieces are:

  • The color names (“maroon”, “orange”) and hex codes
  • vvn_map_style() which is set up for county-level data but works with any sf polygon layer
  • The template scaffolding (create_vvn_story(), create_vvn_dashboard()) which generates VVN-branded files

All other functions — themes, scales, chart helpers, table styling — are geography-agnostic.

Charts

How do I change bar colors?

By default, set_vvn_defaults() makes all bars VT Maroon. To override for a single chart:

# Single color
geom_col(fill = vvn_colors("navy"))

# Color by group (3 groups)
scale_fill_vvn("brand")      # maroon, orange, navy

# Color by group (many groups)
scale_fill_vvn("main")       # up to 10 colors

# Colorblind-safe
scale_fill_vvn("accessible")

How do I change the font size?

Pass base_size to the theme:

theme_vvn(base_size = 14)   # larger — for presentations
theme_vvn(base_size = 10)   # smaller — for small multiples

Or override a specific element:

theme(plot.title = element_text(size = 18))

How do I add a source line?

Use vvn_source() — required on every published VVN figure:

ggplot(df, aes(x, y)) +
  geom_col() +
  vvn_source("U.S. Census Bureau, ACS 5-Year Estimates, 2022")

With a methodological note:

vvn_source("Virginia Employment Commission",
            note = "Data suppressed for localities with fewer than 100 workers.")

How do I make a scatter plot look right?

Scatter plots need grid lines on both axes. Use theme_vvn(grid = "both") or add scatter_grid():

ggplot(df, aes(x, y)) +
  geom_point() +
  theme_vvn(grid = "both")

# Or equivalently:
ggplot(df, aes(x, y)) +
  geom_point() +
  scatter_grid()

How do I make a chart colorblind-safe?

Switch to the accessible palette (Okabe-Ito inspired, 7 colors):

scale_fill_vvn("accessible")
scale_color_vvn("accessible")

Then verify contrast ratios:

vvn_accessibility_check(vvn_palette("accessible"))

VT Maroon (#861F41) and VT Orange (#E5751F) can appear identical under red-green colorblindness. Always differentiate with shape or line type when both appear on the same chart.

How do I save a figure?

vvn_save(my_chart, "figures/chart.png")                        # 8×5 in, 300 dpi
vvn_save(my_chart, "figures/chart.png", width = 6, height = 4)
vvn_save(my_chart, "figures/chart", formats = c("png", "pdf", "svg"))

vvn_save() always uses a white background regardless of your current theme settings.

Maps

What type of map should I use for county-level data?

Use a choropleth (filled polygon) map. Always use a normalized variable (rate, percentage, per-capita) — never a raw count, because larger counties will appear darker just from having more people.

Data type Palette
Continuous, primary metric "maroon_seq"
Continuous, secondary metric "orange_seq"
Above / below average (z-scores, deviations) "diverging"

How do I add county names to a Leaflet map tooltip?

Pass the county name column to label_col:

vvn_map_style(map, data, "rate",
              label_col = "NAME",
              title     = "Rate (%)")

The tooltip will show <County Name>: <value> on hover.

Tables

How do I add a title to my table?

Pass title and optionally subtitle to vvn_table():

gt(df) |>
  vvn_table(
    title       = "Summary Statistics by County",
    subtitle    = "Virginia, 2022 estimates",
    source_note = "Source: U.S. Census Bureau, ACS 5-Year Estimates"
  )

Table titles use headline case; subtitles use sentence case.

How do I highlight significance in a regression table?

Use vvn_table_pvalues() after vvn_table():

gt(tidy(model)) |>
  vvn_table(title = "Regression Results") |>
  vvn_table_pvalues(col_name = "p.value")

By default: p < 0.001 → maroon, p < 0.05 → orange.

Shiny dashboards

How do I deploy my dashboard?

After building with create_vvn_dashboard() and replacing the example data:

# Test locally first
shiny::runApp("my-dashboard")

# Deploy to shinyapps.io
rsconnect::setAccountInfo(name   = "your-name",
                          token  = "YOUR_TOKEN",
                          secret = "YOUR_SECRET")
rsconnect::deployApp("my-dashboard")

How do I connect the Apply Filters button to my charts?

Use bindEvent() so charts only update when the user clicks Apply:

server <- function(input, output, session) {
  filtered_data <- reactive({
    my_data |> filter(year >= input$year_range[1],
                      year <= input$year_range[2])
  }) |> bindEvent(input$apply_btn)   # only recalculate on Apply click

  output$my_chart <- renderPlot({
    ggplot(filtered_data(), aes(x, y)) + geom_col()
  })
}

VVN Insights

How do I render my Insights document?

From the terminal in your project folder:

quarto render index.qmd

Or from R:

quarto::quarto_render("index.qmd")

Make sure embed-resources: true is in your _quarto.yml so the output is a single self-contained HTML file:

format:
  html:
    embed-resources: true

How do I check that my Insights document is complete?

check_vvn_story("my-insight")

This validates that index.qmd contains the required sections (Overview, Background, Findings, Conclusion) and uses theme_vvn() and vvn_source().
